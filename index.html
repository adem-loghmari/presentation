Context

Generic e-commerce breaks on live events

Events are spatial and real-time. Off-the-shelf platforms aren't.

  • Fragmented tools — organizers juggle separate apps for design, sales, and door scanning.
  • No spatial rendering — standard platforms can't draw custom seating charts.
  • Race conditions — a wide seat-lock window causes double-booking, a catastrophic failure mode.
  • Walled gardens — high commissions and no embeddable API to sell on your own site.
Problematic

How do you let an organizer draw a venue once, and let the world book it in real time — without ever double-booking a seat?

Chapter 3 · The Technical Nucleus

The Venue Builder

A production-grade, interactive 2D map editor built on Konva.js inside React 19 — drawing seats, zones, floors and prices that flow straight into the database.

Venue Builder · Performance

Sustaining 60 FPS at stadium scale

A stadium map can hold 5,000+ interactive nodes. Naïve React would freeze. Three disciplines keep it fluid:

📚Layer separation
Static grid, interactive shapes, and ephemeral drawing previews live on independent Konva layers.
👁️Viewport culling
Only shapes inside the current viewport are mounted — the rest carry zero render or hit-test cost.
🧠Memoization
React.memo + useMemo + useCallback so only what changed re-renders.

Orchestrated by 16 specialized hooks — one behavioral concern each (state, drawing, drag-snap, transform, navigation, sync…).

Venue Builder · Smart Tooling

Tools that erase the tedium

🪄Magic-Wand auto-labeling
Sorts selected seats into rows by geometry, then assigns Row A/B/C and seat numbers — 500 seats labeled in under a second.
🤖AI text-to-map
A natural-language prompt → an LLM constrained to a strict JSON schema → a full foundational layout injected into the canvas.
🗺️Minimap & grid snapping
Draggable viewport rectangle for navigation; 10-px grid snap keeps every seat perfectly aligned.
🏢Multi-floor + blueprints
Floor tabs and zone drill-down; trace over an uploaded architectural blueprint, then fade it out.
Canvas → Database

From JSON blob to relational rows

  • The whole canvas serializes to one canvas_data JSON — floors, categories, shapes. Cheap to write while designing.
  • On publish, syncLocations() migrates it in three passes: categories → PriceTiers, shapes → Locations, then LocationPrice links.
  • updateOrCreate by stable IDs — orphaned tiers and locations are pruned automatically.

{ "shapes": [

] }

Reservation Engine

Pessimistic locking kills the race

  • A booking opens an ACID transaction and calls lockForUpdate() on the seat row.
  • MySQL blocks every concurrent request until commit. The second arrival sees the new count and gets 409 Conflict.
  • A held seat becomes a Reservation with a 10-minute expiry, auto-pruned by Laravel's scheduler.
  • Availability is computed with aggregate counts — sold / locked / mine / remaining — in under 100ms.
DB::beginTransaction();
$loc = Location::lockForUpdate()
    ->find($id);

if (held + sold + qty > capacity)
  return 409;

DB::commit();
Real-Time Sync

Everyone sees it at once

SeatUnlocked
hold released / expired

  • Broadcast on a per-timeslot channel via Pusher — the evening show never bleeds into the morning's.
  • Laravel Echo flips a seat available → locked in milliseconds on every connected screen.
  • Client-side cart machine runs the 10-min countdown; "Best Available" auto-picks the best contiguous block.

Security Hardening

Closing the attack surface

  • JWT in localStorage → HTTP-only cookies. The session token becomes physically invisible to JavaScript — the XSS theft vector is gone.
  • CSRF handshake — a csrf-cookie call seeds the XSRF token before every mutating request.
  • Google OAuth 2.0 via Socialite — frictionless sign-in.
  • Embed tokens — a scoped Bearer path for iframe contexts where cookies can't follow.
B2B Developer SDK

GetSeat as a white-label engine

  • API keys scoped to partner:api; shown once, never stored.
  • Five server-to-server endpoints — publish venues, confirm orders, inject pricing.
  • Shadow users: a partner's customer is silently proxied through a managed identity — no GetSeat account needed.
  • Embeddable iframes: drop the reservation portal or full builder into any site.
